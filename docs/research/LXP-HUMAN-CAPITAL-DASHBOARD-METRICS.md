# 調査メモ: LXP 学習ダッシュボード × 経営・人的資本 × スキル開発

コード実装の前提整理用。**仕様書（docs/spec）とは別**に、経営要件のベンチマークを残す。

## ベンチマーク枠組み

- **人的資本可視化指針（改訂版）**（経済産業省・内閣官房・金融庁）— 研修時間・費用、参加率、育成プログラム、効果の開示が論点。
- **ISO 30414** — 人的資本レポーティング。スキル・能力・開発領域と他指標の整合。
- **LXP ベンダー標準的ダッシュボード** — エンゲージメント、完了率、スキル別学習時間・ユニーク学習者、ギャップ。

## 経営がデータを欲しがる主なケース

| ケース | 代表指標の方向性 |
|--------|------------------|
| 人的資本開示・統合報告 | 従業員あたり学習時間、教育費、参加率、主要プログラム |
| 戦略スキル投資（DX/AI 等） | スキルタグ別時間、必須学習達成率、ギャップ推移 |
| コンプライアンス | 必須トレーニング完了率・期限超過 |
| L&D 予算・ROI 対話 | コスト、完了率、評価前後差・バッジ等のプロキシ |

## 学習 → スキル更新を繋ぐ原則

1. コンテンツ・パス・バッジに **スキル ID** を付与する。
2. **修了イベント**をスキル熟練度ルールに渡す（プロトタイプでは `LearnerProgressContext` 等が差し込み口）。
3. 経営画面は「コース完了」だけでなく **スキル領域の供給・ギャップ縮小**で見せる。
