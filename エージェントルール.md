# エージェント運用ルール（Track e-learning プロトタイプ）

このファイルは **AI（Cursor Agent 等）が最初に読むべきリポジトリ固有ルール**です。ユーザー指示や Cursor グローバルルールと矛盾する場合は、**ユーザーの当該指示を優先**します。

## 1. ドキュメントの正規パス

| 種別 | パス | 使い方 |
|------|------|--------|
| 進捗・タスク | `docs/DEVELOPMENT-PROGRESS.md` | 実装スコープ・チェックリストの**唯一の正** |
| マスター仕様 | `docs/spec/00-REBUILD-SPECIFICATION.md` | エントリ。続けて 01→02→03→05→04 の順で読む |
| UI 補足 | `docs/spec/Track e-learning Admin Dashboard UI.md` 等 | 管理画面・受講者 UI の詳細 |
| 調査・リサーチ | `docs/research/` | 仕様と独立した経営・LXP 調査メモ |
| 索引 | `docs/README.md` | 全体の目次 |

**廃止:** `sytem-documentions/`（typo）は存在しません。見かけたら `docs/spec/` または `docs/DEVELOPMENT-PROGRESS.md` に置き換えてください。

## 2. 実装時の行動規範

- **仕様と進捗を先に読む:** いきなり大規模改変せず、`00` と `DEVELOPMENT-PROGRESS.md` でルート・スコープを確認する。
- **変更はタスクに必要な範囲に限定する。** 無関係なリファクタ・ドキュメント量産をしない（ユーザーが明示した場合を除く）。
- **受講者ルート:** 現行は `/learner/skill-hub/*`（Skill Hub）と `/learner/track/*`（Track e-learning）。旧 `/learner/home`・`/learner/skills/*` は `next.config.ts` の redirects で新 URL へ寄せる。新規リンクは新パスだけ使う。
- **認証・レイアウト:** `src/context/AuthContext.tsx`、`src/components/layout/`（AppLayout / Header / Sidebar / SkillHubSidebar / TrackLearnerSidebar 等）を尊重する。
- **ビルド確認:** 変更後は `npm run build` または `./node_modules/.bin/next build` で TypeScript を含めて通す。

## 3. ユーザー向けファイル

- **ゼロから再構築用の長文プロンプト:** `AGENT-PROMPT-PROTOTYPE-DEVELOPMENT.md`（内のパスは `docs/spec/` を指すよう更新済みであること）
- **デプロイ:** `DEPLOY.md`
- **参考メディア:** `ui-screenshots/`（README 参照）

## 4. ドキュメント更新の責務

- 画面やルートを追加・変更したら、**可能な範囲で** `docs/spec/04-PAGES-AND-FEATURES.md` または `00-REBUILD-SPECIFICATION.md` と `DEVELOPMENT-PROGRESS.md` を更新する。
- **プラン用 `.cursor/plans/*.md` はユーザーが編集しない限り触らない**（指示がある場合を除く）。

## 5. 技術スタック（要約）

- Next.js 16（App Router）、React 19、TypeScript、Tailwind CSS 4、`@/*` → `src/*`
- チャート: recharts、アイコン: lucide-react

以上。詳細は常に `docs/spec/` の該当番号ファイルに従うこと。
